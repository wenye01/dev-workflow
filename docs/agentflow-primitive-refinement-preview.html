agentflow 原语细化设计方案

这版方案把上一轮过重的权限和 target 设计收回来：Activation 主要调用 Agent 或 Recipe， Capability 只做轻量运行边界，ContextBuilder 负责输入构建，parallel / pipeline / branch / loop 放到 Workflow 框架层， 人工介入作为一等 workflow 能力处理。

目标不是强安全微内核，而是轻量原语 + Agent 承载 + Recipe 编排权 + ContextBuilder 输入构建 + Workflow Constructs 使用层。

分层模型

不再从原语直接跳到典型场景。先提供框架能力，再覆盖具体 workflow。

Primitive

最小语义基底。

Activation Capability Artifact Event Recipe Policy

Runtime

让原语跑起来的框架模块。

ContextBuilder ActivationRunner ArtifactStore EventLog

Workflow

recipe 作者使用的高层 construct。

parallel pipeline branch loop

Human

人工请求、等待、裁决和恢复。

Agents

Scenarios

建立在 workflow constructs 上的业务流程。

long-running app dev review changes bug hunt

原语调整

Activation target 收敛为 Agent 或 Recipe。aggregation、memory 等不再与 Agent 同级。

Agent Activation

调用一个 Agent 定义。Agent 内部可以运行命令、读写工作区、联网、调用工具，原语层不逐步拦截。

Activation {
  target: { kind: 'agent', ref: 'generator.implementer' },
  context_request,
  expected_outputs,
  capability
}

Recipe Activation

唯一允许产出可执行 Directive 的受控入口。用于动态 plan 展开和嵌套编排。

Activation {
  target: { kind: 'recipe', ref: 'expand-planner-spec' },
  expected_outputs: [{ kind: 'directive' }]
}
旧概念 新归属
system targetAgent 内部自然动作，或 runtime 自动动作。
polling targetexternal.wakeup Event + loop construct。
aggregation targetaggregator.* Agent。
memory targetmemory.summarizer Agent + handoff/summary Artifact。
human target优先作为 approval 机制；需要人类执行时可定义 human.* Agent。

Capability-lite

Capability 是启动前的轻量运行边界，不是文件、网络或工具调用级的安全系统。

工作模式

这次 Activation 大致在做什么。

plan execute review aggregate summarize recipe

可见输入

给 ContextBuilder 的输入选择提示。

task project_index artifacts recent_events handoff_summary

预算与确认

启动前大致判断，完成后记账；预算耗尽后停止后续调度。

max_tokens max_calls max_wall_time approval

Capability 做什么

work_mode
visible_inputs
budget hint
approval before start
runtime_hints

Capability 不做什么

no file permission gate
no network permission gate
no tool-call interception
no mid-activation kill
no destructive-op policy

Runtime 框架

Runtime 是原语到实际执行之间的桥。ContextBuilder 是 Agent 输入构建的核心。

1. Recipe读 state，产出 Activation proposals
2. Policy检查编排来源、预算、loop 限制和 approval
3. ContextContextBuilder 选择 task、Artifact、Event 和 handoff
4. Runner调用 Agent CLI 或 RecipeRunner
5. OutputAgent CLI 返回结构化输出和 usage
6. Record写 Artifact、追加 Event、记录预算
Artifact 是 Activation 的持久输出；输入不是简单等于 Artifact 列表，而是由 ContextBuilder 从 task、project index、selected artifacts、recent events、handoff summary 和 workflow state 中构建。

Workflow Constructs

这些是高层使用能力，不是原语。它们最终 lower 成 Recipe 状态转移和 Activation proposals。

agent()

调用一个 Agent，产出 Artifact。

agent('implement unit', {
  role: 'generator.implementer',
  context,
  output,
  capability
})

parallel()

一次 propose 多个 Activation，等待全部完成。

parallel(units.map(unit =>
  () => agent(`implement ${unit.ref}`)
))

pipeline()

每个 item 独立流过多个 stage，无全局 barrier。

pipeline(units,
  analyze,
  verify,
  summarize
)

branch()

根据 Artifact/Event/WorkflowState 选择下一步。

branch(verdict.status, {
  pass: done,
  fail: fix,
  uncertain: review
})

loop()

支持 fix-until-pass、loop-until-dry、polling loop。

loop({
  until: state => state.pass,
  maxIterations: 3,
  body
})

join()

通常 lower 成 aggregator Agent。

join(reviews, {
  role: 'aggregator.review_panel'
})

requestHuman()

暂停当前 branch，写入人工请求，等待裁决后由 Recipe 分支继续。

requestHuman({
  question: '继续修复还是接受当前结果？',
  options: ['fix', 'accept', 'stop'],
  context
})

编排权限规则

只保留最小编排安全，不做强安全 sandbox。

规则 结果
普通 AgentActivation 产 plan/workflow_spec Artifact 允许，但只是数据。
普通 AgentActivation 产 Directive 忽略或拒绝，不能直接调度。
Recipe / RecipeActivation 产 Directive 允许，经过 schema validation 和 Policy 轻量检查。
预算、loop depth、activation count 超限 停止后续调度，不承诺中途强杀当前 Agent。
Capability.approval.required 启动前请求人工确认。

人工介入

人工介入是关键 workflow 能力，但不是新原语。它通过 Capability、Event、Artifact 和 Recipe branch 组合表达。

1. RequestrequestHuman() 或 Capability.approval 触发请求
2. Record写 human_request Artifact，追加 approval.requested Event
3. Wait当前 workflow branch 进入 waiting_for_human
4. Respond用户给出 approve / reject / choose / revise / stop
5. Persist写 human_decision Artifact，追加 human.responded Event
6. ResumeRecipe 读裁决后分支继续或停止

原语层落点

Capability.approval
Event: approval.requested
Artifact: human_request
Artifact: human_decision
Event: human.responded
Recipe: branch(decision)

裁决结果

approve -> resume pending activation
reject  -> skip or fallback
choose  -> run selected branch
revise  -> feed notes to planner/generator
stop    -> stop workflow

场景预览

典型场景建立在 constructs 上，不直接侵入原语层。

Planner 
const plan = await agent('plan task', {
  role: 'planner.initial',
  context: context({ mode: 'task', include: { task: true, project_index: true } }),
  output: { kind: 'planner_package', schema: PLANNER_PACKAGE },
  capability: { work_mode: 'plan', budget: { max_tokens: 40000 } }
});
Generator / Evaluator Loop 
await pipeline(plan.units,
  unit => loop({
    maxIterations: unit.max_fix_rounds + 1,
    body: async state => {
      const change = await agent('implement or fix', { role: 'generator.implementer' });
      const verdict = await agent('evaluate', { role: 'evaluator.initial', input: change });
      return verdict.status === 'pass' ? done(verdict) : continueLoop({ verdict });
    }
  })
);
Optional Review Escalation 
await branch(unit.status, {
  pass: done,
  fail_after_retries: () => parallel([
    () => agent('security review', { role: 'reviewer.security' }),
    () => agent('correctness review', { role: 'reviewer.correctness' }),
    () => agent('adversarial critique', { role: 'critic.adversary' })
  ]).then(reviews => join(reviews, { role: 'aggregator.review_panel' }))
});
